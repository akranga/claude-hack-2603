01 / 10 — ~30s

👋 Introduce Yourself to Claude

  • Tell Claude about yourself, your role, your project, your preferences
  • Claude stores it in local auto memory — a MEMORY.md file
  • Next session: Claude already knows your habits, tooling, and conventions
  • It learns from your corrections over time — improving with every session

02 / 10 — ~60s

📝 Context Engineering: CLAUDE.md First

  • Run /init — generates a starter from your project
  • Keep it under 200 lines — bloated = ignored
  • Only what Claude gets wrong without it: build commands, style, conventions
  • Prune ruthlessly — if Claude already does it right, delete the rule

03 / 10 — ~45s

⚡ Skills + CLI Tools over MCPs

📚

Skills

Domain knowledge & reusable workflows.
Zero context cost when not invoked.

🔌

MCPs

Live tool connections: Figma, databases, Playwright.

⌨

CLI Tools

Most context-efficient way to talk to external services. gh, aws, gcloud

  • Install skills across teams with Skillfish: npx skillfish add owner/repo
  • Wrap skills into a plugin for org-wide distribution & versioning
  • Meta-skill: use a skill-builder skill to create new skills
  • Claude can learn unfamiliar CLIs from --help
Skills for knowledge. MCPs for live connections. CLI for everything else.
04 / 10 — ~45s

🤖 Subagents: Specialized Workers

  • Lives in .claude/agents/
  • Runs in its own context window
  • Has its own tools, model & permissions
  • Auto-delegates or invoke with @agent-name
  • Use for code review — catches mechanical issues in PRs
Examples:
code-reviewer (read-only, Sonnet)   debugger (Edit, Opus)   security-scanner (Haiku)

Distill repeatable patterns hereWhen you need isolated execution
When subagents repeat the same patterns → distill into a Skill. That's where compounding happens.
05 / 10 — ~30s

🔍 /review — Code Review with Claude

  • /review — built-in command, analyzes changes instantly
  • Create a review subagent — read-only, Sonnet, own context
  • Claude catches mechanical issues: style, bugs, security
  • You focus on design and intent — what Claude can't judge
  • Run in CI/CD: claude --print -p "Review this PR"

🌟 #1 Leverage Point from Anthropic
Set up conditions where Claude can verify its own work — run tests, check emulator output, inspect logs, compare screenshots.
The more Claude verifies autonomously, the less you become the bottleneck.
  • Hooks: pre/post command hooks in .claude/hooks/ — guaranteed checks on every change
  • Use /rewind for checkpoints. After 2 failed corrections → /clear and start fresh
08 / 10 — ~30s

🧠 Manage Context Aggressively

  • Reasoning budget: "think", "think hard", "ultrathink" — use /effort to adjust
  • Use /model to switch mid-session as task complexity changes
Right model + right reasoning depth = optimal cost & quality.
10 / 10 — ~20s

📡 Claude Remote Control

  • Start a task on your laptop, continue from anywhere — phone, tablet, web
  • Kick off long-running tasks and walk away
  • Review results & approve permissions remotely
  • Pair with agent teams for multi-session orchestration
